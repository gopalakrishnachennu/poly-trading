# Engineering Constitution

1. Financial safety invariants outrank profit objectives.
2. Durable inputs must reproduce deterministic state.
3. State ownership must be explicit and single-writer.
4. Unknown or unreconciled state cannot increase exposure.
5. External inputs are untrusted and bounded at every parser.
6. Failure behavior is specified and tested before live use.
7. Infrastructure is introduced only for a measured requirement.
8. Architecture decisions are versioned in Git.
